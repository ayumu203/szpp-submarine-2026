# 潜水艦撃沈ゲームに関して

## ルール
- 以下でゲームがプレイできる(昔のSZPPerが作成したもの).
- [潜水艦撃沈ゲーム Visualizer](https://github.com/szpp-dev-team/submarine-war-visualizer)
- 一応のサンプルが`feat/codex`にあるけどあまり参考にしないで.

### ゲーム空間

- $5 \times 5$のマスに潜水艦を$4隻$配置.
- $x方向$を左側から$1, 2, 3, 4, 5$と, $y方向$を$A, B, C, D, E$と表現(読みはそれぞれアルファ, ベータ, チャーリー, デルタ, エコー).
- それぞれの潜水艦はHP3を持ち攻撃を受けるとHPが$1$減る.
- HPが0となった潜水艦を撃沈状態とする.

### ゲームの流れ

1. 潜水艦を配置し, 先行・後攻を決定する.
1. 先行のプレイヤーは"攻撃"または"移動"を宣言する.
1. （攻撃の場合）後攻プレイヤーは"攻撃"の結果を報告する.
1. 後攻プレイヤーも同様の手順で操作を行う.
1. 20分経過時点で残存HPの多い方を勝者とする.

### 攻撃

- 撃沈状態でない潜水艦の周囲8マスを範囲として, その中から1マスを選び魚雷を発射することができる.
- 魚雷が命中した場合には潜水艦のHPを1減らす.
- HPが0となった潜水艦は撃沈状態となる.
- 自軍の潜水艦へは攻撃ができない.

### 移動

- 撃沈状態でない潜水艦は自身の上下左右のいずれかの2マス以内のマスへ移動することができる.
- ただし撃沈状態の潜水艦が存在するマス, 撃沈状態の潜水艦を超えるような移動はできない.

### 宣言と報告

- 攻撃:
    - 宣言: "C-3に魚雷発射."
    - 報告: （命中した場合）"命中.",（命中し撃沈した場合）"命中撃沈", （周囲8マスに潜水艦がいる場合）"波高し."
- 移動: 
    - 宣言: "潜水艦を東側に2マス移動."

## 開発システムの概要
上記ルールに基づくゲームを作成する.

### つくるもの
- ユーザの操作画面.
- ユーザの入力を受取, 処理してUpstashへ保存するサーバ.
- 入力・保存情報をもとに思考するCPU Player.

### フロントエンド

- HTML + JavaScript + jQuery(jQuery Core 3.7.1 minified)

### バックエンド

- Go(1.22.2)
- 標準ライブラリのみで問題ないと思う.

### DB

- Upstash
- Redis を扱えるサーバレスのデータベース.


### システムのアーキテクチャ構成
```
| フロントエンド | ー(HTTP Request)→ |バックエンド| ー(HTTP Request)→ | Upstash |
```
